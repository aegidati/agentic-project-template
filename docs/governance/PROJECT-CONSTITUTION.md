# Project Constitution

## Purpose
Define the foundational principles and governance model for this agentic development project.

## Core Principles
- **Stack-agnostic design**: Architecture decisions are deferred to the derived project.
- **Documentation-driven development**: Features are documented before implementation.
- **ADR-based governance**: Architecture and significant decisions are recorded in ADRs.
- **Agent-assisted workflow**: Agents enforce documentation discipline and process compliance.

## Project Scope
- This is a **template repository** for agentic development.
- It provides governance structure, not application code.
- Derived projects must install a starter pack to add a specific architecture.

## Governance Model
- All decisions are documented in [AGENTIC-WORKFLOW.md](./AGENTIC-WORKFLOW.md).
- Feature lifecycle is tracked in `docs/features/<FEATURE-NAME>/`.
- Architectural decisions are recorded in `docs/adr/`.
- Process changes are managed via [CHANGE-MANAGEMENT.md](./CHANGE-MANAGEMENT.md).
